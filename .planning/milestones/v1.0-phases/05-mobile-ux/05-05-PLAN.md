<objective>
Fix UAT gap: selecting an autocomplete suggestion with category/quantity data does not auto-expand the "More details" panel, making the populated fields invisible to the user.

Purpose: The handleSuggestionSelect function sets category and quantity state but never calls setExpanded(true), so the user cannot see those fields were populated. This is a major usability gap — the whole point of autocomplete pre-filling is lost if the fields are hidden.

Output: Patched AddItemBar.tsx with auto-expand behavior and a regression test in AddItemBar.test.tsx.
</objective>
