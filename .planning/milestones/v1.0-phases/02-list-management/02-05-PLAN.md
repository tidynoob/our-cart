<objective>
Fix the inline edit category Select dropdown bug where opening the dropdown or clicking a category option exits edit mode prematurely.

Purpose: UAT test 6 identified a major issue -- users cannot edit an item's category because the Select Portal renders outside the row's DOM tree. When the Select opens (or the user clicks a SelectItem), focus moves to the portal element. The handleRowBlur setTimeout(0) check finds document.activeElement outside rowRef, fires handleSave(), and exits edit mode before the user can interact with the dropdown. This blocks the "edit category" part of LIST-02 (edit an existing item's category).

Output: A working ItemRow where the category Select opens, stays open, accepts a selection, and only exits edit mode when focus genuinely leaves the row (not to the Select portal).
</objective>

  <action>
    ROOT CAUSE: SelectContent wraps its children in SelectPrimitive.Portal (select.tsx line 72), rendering the dropdown outside the rowRef DOM tree. When the Select opens or user clicks a SelectItem, focus transfers to the portal. handleRowBlur's setTimeout(0) check runs rowRef.current.contains(document.activeElement), finds false (activeElement is in the portal, not the row), and calls handleSave() which exits edit mode.

    FIX in src/components/ItemRow.tsx:

    1. Add a ref to track Select open state:
       const selectOpenRef = useRef(false)

    2. Add an onOpenChange callback to the Select component (line 168). The @base-ui/react Select.Root accepts onOpenChange as a prop with signature (open: boolean, eventDetails) => void. Add:
       onOpenChange={(open) => { selectOpenRef.current = open }}

       This sets selectOpenRef.current to true when the dropdown opens and false when it closes.

    3. Modify handleRowBlur (lines 105-111) to skip save when the Select is open. Change the condition inside the setTimeout from:
       if (rowRef.current && !rowRef.current.contains(document.activeElement)) { handleSave() }
       to:
       if (selectOpenRef.current) return;
       if (rowRef.current && !rowRef.current.contains(document.activeElement)) { handleSave() }

       The early return when selectOpenRef.current is true means: "The Select portal has focus, this is expected, do not exit edit mode."

    4. Keep the existing onMouseDown={(e) => e.preventDefault()} on SelectTrigger (line 175). This prevents blur when clicking the trigger itself. The onOpenChange fix handles the portal focus case which onMouseDown cannot prevent.

    Do NOT modify src/components/ui/select.tsx -- the Portal is correct behavior for a Select dropdown. The fix belongs entirely in ItemRow's blur logic.

    Do NOT remove or change any other blur-handling logic (the onMouseDown preventDefault on trash button, cancel button, etc.). Those are correct for their elements.
