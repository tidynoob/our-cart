## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | oklch(1 0 0) / --background | Page background, list area |
| Secondary (30%) | oklch(0.97 0 0) / --secondary | Category section headers, input backgrounds |
| Accent (10%) | oklch(0.205 0 0) / --primary | Add button, save confirmation icon |
| Destructive | oklch(0.577 0.245 27.325) / --destructive | Delete confirmation, trash icon active state |

Accent reserved for: Add item button fill, save-success checkmark only

### Attribution Colors (2-person scope)

| Person Slot | Badge Background | Text Color | Derivation |
|-------------|-----------------|------------|------------|
| Person A | oklch(0.75 0.15 250) | oklch(0.30 0.10 250) | Blue tone — deterministic from name hash even slots |
| Person B | oklch(0.80 0.15 150) | oklch(0.35 0.10 150) | Green tone — deterministic from name hash odd slots |

Color assignment: `charCodeAt(0) % 2` on the stored name → slot A or B. Same name always gets same color. Both pass WCAG AA contrast on white background.

## Interaction Patterns

| Interaction | Behavior |
|-------------|----------|
| Add item (Enter) | Submit name-only item → clear input → item appears at top of its category |
| Add item (expanded) | Submit with qty + category → collapse expanded area → clear all fields |
| Edit item | Tap row → inline edit → blur/Enter saves → exit edit mode |
| Delete item | Tap trash in edit mode → confirmation row → confirm deletes → row animates out |
| Category dropdown | Native select on mobile (OS picker), styled dropdown on desktop |

## Responsive Behavior

| Breakpoint | Behavior |
|------------|----------|
| < 640px (mobile default) | Full-width layout, Add button icon-only, native select for category |
| >= 640px (sm) | max-w-md centered, Add button shows "Add" label, styled dropdown |
