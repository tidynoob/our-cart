# Requirements Archive: Our Cart v1.0

**Archived:** 2026-05-26
**Milestone:** v1.0 MVP
**Core Value:** Two people can see the same grocery list update in real-time, so nothing gets missed or double-bought.

## v1 Requirements — All Complete

### List Management

- [x] **LIST-01**: User can add item with name (required), quantity (optional), and category (optional) — Phase 2
- [x] **LIST-02**: User can edit an existing item's name, quantity, or category — Phase 2
- [x] **LIST-03**: User can delete an item from the list — Phase 2
- [x] **LIST-04**: Items display who added them (initials or color indicator per device/person) — Phase 2
- [x] **LIST-05**: Previously added items appear as autocomplete suggestions during entry — Phase 5
- [x] **LIST-06**: Items auto-sort by category in the list view (Produce, Dairy, etc.) — Phase 2

### Real-Time Sync

- [x] **SYNC-01**: Changes (add, edit, check off, delete, clear) appear on both devices within 2 seconds — Phase 4
- [x] **SYNC-02**: App reconnects and re-fetches after WebSocket disconnection (e.g., screen lock on mobile) — Phase 4
- [x] **SYNC-03**: Connection status indicator shows whether sync is active — Phase 4

### Sharing

- [x] **SHARE-01**: User can create a new list and receive a shareable link or code — Phase 1
- [x] **SHARE-02**: Partner can join the list by opening the shared link — no account creation needed — Phase 1
- [x] **SHARE-03**: Shared link acts as the only access credential (URL contains list ID) — Phase 1

### Shopping Flow

- [x] **SHOP-01**: User can check off an item (item stays visible, shown crossed out/dimmed) — Phase 3
- [x] **SHOP-02**: User can uncheck a previously checked item — Phase 3
- [x] **SHOP-03**: User can clear all checked items (removes from both views) — Phase 3
- [x] **SHOP-04**: Clear action requires confirmation before executing — Phase 3

### Mobile UX

- [x] **UX-01**: Layout is phone-first and responsive (works on desktop too) — Phase 5
- [x] **UX-02**: Tap targets are large enough for one-handed use while walking (min 44px) — Phase 5
- [x] **UX-03**: Adding an item takes fewer than 3 taps from app open — Phase 5
