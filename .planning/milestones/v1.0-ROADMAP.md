# Milestone v1.0: MVP

**Status:** SHIPPED 2026-05-26
**Phases:** 1-5
**Total Plans:** 19

## Overview

Our Cart ships in five phases that mirror the dependency graph of the app itself: stand up the data layer and shareable link first, then build list CRUD, then the shopping check-off flow, then layer real-time sync on top of proven mutations, and finally harden the mobile experience. Every phase delivers a coherent, independently testable capability.

## Milestone Summary

**Key Decisions:**

- No auth — shared URL link is the only credential (minimize friction for 2-person use)
- Real-time sync over polling (core value depends on instant updates)
- Check-off-then-clear flow (both people see what was bought before clearing)
- Single list for v1 (validate core value first)
- Subscribe-before-fetch pattern for Supabase Realtime
- Optimistic mutations with per-item rollback for responsive UX
- Belt-and-suspenders offline detection (window events + mutation error guards)

**Issues Resolved:**

- NamePromptDialog Save button not working (@base-ui/react type override bug — switched to native button)
- Category Select portal blur exiting edit mode (portal-aware blur guard)
- SyncStatus not showing Reconnecting on brief network drops (added mutation-offline guards)
- Autocomplete not auto-expanding details panel (added setExpanded on suggestion select)
- TypeScript build errors (noUnusedLocals, mock types)

**Issues Deferred:**

- Supabase keep-alive for 7-day inactivity pause (OPS-01, v2)
- Presence indicator showing partner is viewing (OPS-02, v2)
- Item notes field (ITEM-01, v2)
- Manual item reorder within category (ITEM-02, v2)

**Technical Debt Incurred:**

- None significant — v1 is clean
