## Cross-Phase Integration

Only Phase 6 exists, so there is no v2.0 phase-to-phase wiring to verify yet. The integration check instead verified Phase 6's auth layer lands on the existing v1.0 app without regression:

- **ProtectedRoute on `/list/:code`** — does not break v1.0 list-view / item-CRUD; `isLoading` guard prevents login-flash. WIRED.
- **PKCE auth + realtime worker** — coexist as sibling keys in one `createClient`; real-time sync (v1.0 core value) intact. WIRED.
- **`realtime.setAuth()` on auth change** — rotates JWT only, does not tear down item/list channels. WIRED.
- **LandingPage refactor** — still renders CreateListForm / JoinListForm for authenticated users. WIRED.
- **RLS backward-compat** — `IS NULL OR (select auth.uid()) = …` on all four verbs preserves access to legacy v1.0 anonymous rows. WIRED.

**No regression blockers.**

## Tech Debt

**Phase 6 — owner_id default asymmetry (1 item).** `items_auth.sql` sets `DEFAULT auth.uid()` on `items.user_id`; `lists_auth.sql` sets no default on `lists.owner_id`. An authenticated user creating a list through the unchanged v1.0 `CreateListForm` (which omits `owner_id`) produces a list with `owner_id = NULL`. It passes RLS today via the anonymous branch, so nothing breaks now — but Phase 7 (Lists Infrastructure, LIST-*) introduces list ownership and must set `owner_id` explicitly on insert or add a column default. Flag carried forward to Phase 7.

## Bottom Line

Phase 6 is solid and ready to build on. The milestone simply isn't done — phases 7, 8, 9, 10 remain. Continue execution; do not archive v2.0 yet.
