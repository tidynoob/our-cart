<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| sessionStorage → navigate() | returnTo path is read from sessionStorage; navigate() uses it to redirect |
| signInWithGoogle() error → LoginPage error prop | OAuth error message surfaced to user for retry |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-09 | Tampering | Open redirect via sessionStorage returnTo manipulation | Mitigate | returnTo is only used within the same origin; navigate() from react-router-dom performs client-side navigation only — cannot redirect to an external URL. External URLs would be ignored by the router's path matching. |
| T-06-10 | Information Disclosure | OAuth error message displayed to user | Accept | Error messages from Supabase Auth (e.g., "Email not confirmed") are informational; no secrets exposed. User can retry via the same button. |
| T-06-SC | Tampering | npm/pip/cargo installs | Accept | No new package installs in this plan (RESEARCH.md confirms no new dependencies for this phase). |
</threat_model>
