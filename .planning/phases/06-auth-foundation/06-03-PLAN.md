<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser render → ProtectedRoute | User can navigate directly to /list/:code — ProtectedRoute is a UX guard, not a security control (RLS is the enforcement layer) |
| App mount → onAuthStateChange subscription | Subscription must be cleaned up on unmount to prevent memory leaks |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-06 | Elevation of Privilege | Direct API call bypassing ProtectedRoute | Accept | ProtectedRoute is UX only; Supabase RLS policies enforce access at the database layer. Client-side route guards do not prevent direct API calls. |
| T-06-07 | Denial of Service | onAuthStateChange subscription leak (missing cleanup) | Mitigate | useEffect in App.tsx returns the cleanup function from initialize(); React calls cleanup on unmount. |
| T-06-08 | Information Disclosure | returnTo URL stored in sessionStorage | Accept | sessionStorage is scoped to the tab/origin; the path stored is the intended navigation destination, not a secret. Cleared immediately after redirect. |
| T-06-SC | Tampering | npm/pip/cargo installs | Accept | No new package installs in this plan (RESEARCH.md confirms no new dependencies for this phase). |
</threat_model>
