<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User input (list name, join code) → Supabase | All user-entered text must be validated and sanitized before DB operations |
| Supabase response → UI render | Error messages from Supabase must never reach the UI |
| Web Share API / Clipboard API → Browser | Both require HTTPS and user gesture; feature-detect both |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering | List name field — XSS via malicious list name | Mitigate | React JSX escapes all interpolated strings; list name rendered in `<h1>` and `ShareBanner` as text — no `dangerouslySetInnerHTML` anywhere in this plan. |
| T-03-02 | Information Disclosure | Supabase INSERT error exposed in CreateListForm | Mitigate | CreateListForm renders only `'Could not create list. Please try again.'` on error — never `error.message` from Supabase (V7). Verified in CreateListForm.test.tsx. |
| T-03-03 | Elevation of Privilege | JoinListForm accepts malformed share code and queries Supabase | Mitigate | JoinListForm validates extracted code with `/^[A-Za-z0-9_-]{8}$/` before navigating (V5). ListPage still handles not-found gracefully even if navigation occurs with invalid code. |
| T-03-04 | Denial of Service | Clipboard API / Web Share API called without HTTPS or user gesture | Mitigate | Copy button calls `navigator.clipboard.writeText` (requires HTTPS — Vercel always serves HTTPS). Share button guarded by `if (navigator.share)` and called directly in onClick handler (Pitfall 4). |
  </threat_model>
