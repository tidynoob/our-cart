## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing, gap-2 |
| md | 16px | Default element spacing, p-4, gap-4 |
| lg | 24px | Section padding, gap-6 |
| xl | 32px | Layout gaps |
| 2xl | 48px | Major section breaks, min-h-[48px] tap targets |
| 3xl | 64px | Page-level spacing |

Exceptions:
- **44px:** Minimum tap target for interactive elements per UX-02 / WCAG 2.5.8. Used as `min-h-[44px]` and `min-w-[44px]` on checkbox wrappers (already implemented), category select triggers (D-12), and "more details" toggle (D-14). Not a spacing token but a sizing constraint.

## Typography

| Role | Size | Weight | Line Height | Tailwind Class |
|------|------|--------|-------------|----------------|
| Body | 16px | 400 (regular) | 1.5 | `text-base font-normal leading-normal` |
| Label / Meta | 14px | 400 (regular) | 1.5 | `text-sm font-normal` |
| Heading | 24px | 600 (semibold) | 1.2 | `text-2xl font-semibold` |
| Status | 12px | 400 (regular) | 1.5 | `text-xs font-normal` |

**Constraints:**
- All `<input>` and `<textarea>` elements MUST use `text-base` (16px) minimum to prevent iOS Safari auto-zoom on focus. Already enforced on existing inputs.
- Autocomplete suggestion items use `text-base` (16px) for the item name and `text-sm` (14px) for the category hint.
- Category section headers use `text-xs font-semibold uppercase tracking-wide` (existing pattern).

**Source:** Existing codebase patterns in AddItemBar, ItemRow, ListPage, CategorySection, SyncStatus.

## Color

| Role | Value (oklch) | CSS Variable | Usage |
|------|---------------|-------------|-------|
| Dominant (60%) | `oklch(1 0 0)` | `--background` | Page background, surfaces |
| Secondary (30%) | `oklch(0.97 0 0)` | `--secondary` | Cards, category headers, hover/active states, autocomplete dropdown background |
| Accent (10%) | `oklch(0.205 0 0)` | `--primary` | See reserved-for list below |
| Destructive | `oklch(0.577 0.245 27.325)` | `--destructive` | Delete confirmation, "Clear Items" button |

**Accent reserved for:**
1. Add button (primary variant) in AddItemBar
2. Checked checkbox fill (`data-[checked]:bg-primary`)
3. "Create List" / "Join" buttons on LandingPage
4. Focus ring (`--ring`) on inputs and interactive elements

**Muted foreground** (`oklch(0.556 0 0)` / `--muted-foreground`): Used for placeholder text, category header text, quantity text, "More details" toggle, SyncStatus connecting state label.

**Status colors (functional, not decorative):**
- Green: `text-green-600` / `bg-green-500` -- SyncStatus "Live" indicator only
- Amber: `text-amber-600` / `bg-amber-500` -- SyncStatus "Connecting" / "Reconnecting" indicator only

**Theme color meta tag (D-08):** `#ffffff` (matches `--background` in light mode, oklch(1 0 0)). Applied as `<meta name="theme-color" content="#ffffff">`.

**Source:** `src/index.css` :root block, existing component color usage.

## Component Inventory (Phase 5 New + Modified)

### New: Autocomplete Suggestion Dropdown

**Purpose:** Shows matching previously-added items below the name input in AddItemBar (LIST-05).

| Property | Value |
|----------|-------|
| Position | Absolutely positioned below name input, full width of input |
| Background | `bg-popover` (oklch(1 0 0)) |
| Border | `border border-border rounded-lg shadow-md` |
| Max items visible | 5-8 suggestions (D-03) |
| Max height | `max-h-[280px]` with `overflow-y-auto` |
| Z-index | `z-50` (matches SelectContent pattern) |
| Animation | None -- instant show/hide for speed |

**Suggestion Item:**

| Property | Value |
|----------|-------|
| Height | `min-h-[44px]` (UX-02 tap target compliance) |
| Padding | `px-3 py-2` |
| Layout | Flex row: item name (text-base, flex-1) + category badge (text-sm, text-muted-foreground) |
| Hover / Focus | `bg-accent text-accent-foreground` (matches SelectItem pattern) |
| Active | `bg-accent` with visual press feedback |
| Keyboard | Arrow up/down to navigate, Enter to select, Escape to dismiss |
| Separator | `border-b border-border` between items (not on last item) |

**Behavior (from D-01 through D-04):**
- Source: `SELECT DISTINCT name, category, quantity FROM items WHERE list_id = ?` on mount, cached in local state
- Filter: Case-insensitive prefix match on cached list as user types (D-02)
- Selection populates name field, pre-fills category and quantity from most recent match (D-03)
- Selection does NOT auto-submit -- user confirms with Add button or Enter (D-04)
- Dismiss: blur, Escape key, or selection (Claude's Discretion)
- Debounce: Not required -- local filter on small cached list (Claude's Discretion)

### Modified: AddItemBar

4. **Autocomplete dropdown:** Renders as a sibling below the name input row, within the form. Positioned with `relative` parent on the input container.

## Interaction Contracts

### Autocomplete Keyboard Interaction

| Key | Action |
|-----|--------|
| Type characters | Filter suggestions by prefix, show dropdown if matches exist |
| Arrow Down | Move focus to next suggestion (wraps to first) |
| Arrow Up | Move focus to previous suggestion (wraps to last) |
| Enter (dropdown open, item focused) | Select focused suggestion, close dropdown |
| Enter (dropdown open, no focus) | Submit form as normal (existing behavior) |
| Enter (dropdown closed) | Submit form as normal (existing behavior) |
| Escape | Close dropdown, keep typed text |
| Tab | Close dropdown, move focus to next form element |

### Autocomplete Touch Interaction

| Gesture | Action |
|---------|--------|
| Tap input | Focus input, show keyboard, show dropdown if text matches (D-05: no auto-focus on page load) |
| Type on keyboard | Filter suggestions in real-time |
| Tap suggestion | Populate name/category/quantity fields, close dropdown (D-04: no auto-submit) |
| Tap outside dropdown | Close dropdown, keep typed text |

### Apple Touch Icon (D-08)

Generate a 180x180 PNG from the existing `favicon.svg` (purple lightning bolt). This is the standard size for iOS home screen bookmarks. Place at `public/apple-touch-icon.png`.

**Claude's Discretion resolution:** Adapt the existing `favicon.svg` rather than creating a new icon. The purple lightning bolt is the established brand mark.

### Vercel SPA Rewrite (D-09)

Create `vercel.json` at project root:

```json
{
  "rewrites": [
    { "source": "/(.*)", "destination": "/index.html" }
  ]
}
```

This ensures `/list/:code` routes work on direct URL access and page refresh.

### PWA (D-10)

No PWA manifest or service worker. Explicitly out of scope per CONTEXT.md D-10 and REQUIREMENTS.md "Out of Scope" table.

## Accessibility Contract

| Requirement | Implementation |
|-------------|---------------|
| Autocomplete ARIA | `role="combobox"` on input, `aria-expanded`, `aria-controls` pointing to listbox ID, `aria-activedescendant` for focused option |

| Tap targets (WCAG 2.5.8) | All interactive elements meet 44px minimum (UX-02) |
| iOS zoom prevention | `text-base` (16px) on all inputs (existing pattern, maintained) |
| Focus visible | Ring style via `--ring` token on all focusable elements (existing) |
| Keyboard navigation | Full arrow key + Enter + Escape support for autocomplete |
