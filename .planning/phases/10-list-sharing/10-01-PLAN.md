<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| PostgREST → SECURITY DEFINER functions | Authenticated callers invoke is_list_member and redeem_invite via PostgREST RPC; functions run with elevated DB privilege (bypass RLS) |
| anon role → list_members INSERT | Direct insert must be blocked; only redeem_invite fn may insert |
| share_code parameter → lists lookup | p_share_code text input from untrusted client passed to SQL function |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-01 | Information Disclosure | items/lists SELECT RLS | mitigate | is_list_member helper + owner-OR-member policies ensure non-members get zero rows; non-member SELECT returns empty (RLS blocks). Verified by acceptance criteria on items_select + lists_select. |
| T-10-02 | Elevation of Privilege | redeem_invite SECURITY DEFINER | mitigate | GRANT EXECUTE TO authenticated only (anon blocked); auth.uid() in INSERT ensures membership only for the JWT caller — never an attacker-supplied user_id; ON CONFLICT DO NOTHING prevents injection of duplicate rows. |
| T-10-03 | Elevation of Privilege | is_list_member SECURITY DEFINER | mitigate | SET search_path = '' on both SECURITY DEFINER functions prevents search_path injection; GRANT EXECUTE TO authenticated only. |
| T-10-04 | Information Disclosure | share_code guessability | accept | nanoid(8) = 64^8 ≈ 281 trillion combinations. Acceptable for 2-user household app with no rate-limit requirement. Not acceptable for public app — documented residual risk. |
| T-10-05 | Tampering | list_members INSERT via REST | mitigate | No INSERT policy on list_members — direct REST INSERT denied for all roles. Only redeem_invite (SECURITY DEFINER, authenticated only) may insert. |
| T-10-06 | Tampering | RLS recursion (list_members ↔ lists) | mitigate | list_members SELECT policy uses only direct column check (user_id = auth.uid() OR owner subquery on lists.owner_id) — does NOT call is_list_member. is_list_member only called from lists/items policies. No cycle. |
| T-10-SC | Tampering | npm/pip/cargo installs | accept | No new packages this phase per RESEARCH.md audit |
</threat_model>
