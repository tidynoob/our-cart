# Phase 8 — UI Design Contract

> Visual and interaction contract for the App Shell & Sidebar phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
> Scope: slide-in left-anchored navigation drawer (NAV-01) + active-list highlight (NAV-02). Built on the existing `@base-ui/react` Dialog primitive. Zero new dependencies. Phone-first, one overlay layout at all breakpoints.

## Spacing Scale

Declared values (Tailwind v4 4px-base scale; multiples of 4 only):

| Token | Value | Usage in this phase |
|-------|-------|---------------------|
| xs | 4px (`gap-1`) | Icon-to-label gaps, trigger-icon button internal |
| sm | 8px (`gap-2`, `py-2`) | Drawer nav vertical padding, header control gaps |
| md-touch | 12px (`py-3`, `gap-3`) | Touch-target row padding — grid-aligned (4×3), matches `LandingPage` rows |
| md | 16px (`p-4`) | Drawer header padding, row horizontal padding (`px-4`) |
| lg | 24px | Reserved — not required this phase |
| xl | 32px | Reserved — not required this phase |

Exceptions:
- **Touch targets:** list rows and the hamburger trigger MUST be ≥48px tall. Rows use `min-h-[48px]` (matching `LandingPage` rows) with `py-3` (12px) vertical padding inside `px-4`. The trigger uses the existing button `size="icon"` (`size-8` = 32px visual) but its row container guarantees a ≥48px tap area, OR use `h-10 w-10`/`min-h-[44px]` if standalone. 12px (`py-3` / `gap-3`) is a permitted non-8pt step already established in `LandingPage`.

## Typography

All from Geist Variable. The codebase uses only two weights in this surface area; heading rows reuse `font-heading` (= `font-sans`). Declared sizes for THIS phase only:

| Role | Size | Weight | Line Height | Token |
|------|------|--------|-------------|-------|
| Drawer title ("Your Lists") | 16px | 600 (semibold) | leading-none (~1.0) | `font-heading text-base font-semibold leading-none` |
| List row label (default) | 14px | 400 (normal) | ~1.4 | `text-sm font-normal` — mirrors `LandingPage` rows |
| List row label (active) | 14px | 600 (semibold) | ~1.4 | `text-sm font-semibold` — active-state weight bump (NAV-02) |
| Empty-state / muted text | 14px | 400 (normal) | ~1.4 | `text-sm text-muted-foreground` |

Weights: regular (400) + semibold (600) — exactly two. The drawer title and the active row both use 600; everything else uses 400, so the active-row weight bump (400 → 600) is the single hierarchy step. Body line-height for the read-only nav is the Tailwind `text-sm` default (~1.43); headings `leading-none` to match the existing `DialogTitle`. Do not introduce new font sizes beyond `text-base`/`text-sm`.

## Color

Neutral/grayscale OKLCH palette (no chromatic accent — destructive red is the only hue). Use the dedicated `--sidebar*` tokens for the drawer surface; they exist precisely for this component and keep it visually separable from page content.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `bg-background` (`oklch(1 0 0)` light) | Drawer panel surface, page behind |
| Secondary (30%) | `bg-sidebar` + `text-sidebar-foreground` | Drawer container surface (preferred over plain `bg-background` to delineate the nav); `border-sidebar-border` for the header divider |
| Accent (10%) | `bg-sidebar-accent` / `bg-accent` (`oklch(0.97 0 0)`) | RESERVED — active list row background + row hover only |
| Destructive | `text-destructive` (`oklch(0.577 0.245 27.325)`) | Not used in this phase (no destructive actions in the sidebar) |

Backdrop: `bg-black/10` + `supports-backdrop-filter:backdrop-blur-xs` (exact match to existing `DialogOverlay`).
Drawer edge: `ring-1 ring-foreground/10` + `shadow-lg` (ring matches existing dialog; shadow lifts the overlay off content).

Accent reserved for: (1) the active list row background (`bg-sidebar-accent`), (2) list-row hover background (`hover:bg-sidebar-accent` / `hover:bg-accent`). Never apply accent to the drawer title, the trigger, or non-row chrome. No new hex/oklch values — semantic tokens only.

## Component Contracts

Prescriptive specs the executor implements against. All classes are Tailwind v4 utilities already in use in the codebase.

### Drawer container (Sidebar → `Dialog.Popup`)
- Position: `fixed inset-y-0 left-0 z-50` (left-anchored; overrides centered `DialogContent` — do NOT reuse `DialogContent`).
- Height: `h-dvh` (dynamic viewport — avoids iOS Safari `100vh` cutoff). Fallback `inset-y-0`.
- Width: `w-72` (288px) capped by `max-w-[80vw]` so it never spans a wide desktop viewport (D-06). One layout at all breakpoints — no `md:` rail variant.
- Surface: `bg-sidebar text-sidebar-foreground flex flex-col`.
- Edge: `shadow-lg ring-1 ring-foreground/10`. Radius: square left edge; optionally `rounded-r-xl` only if it reads cleanly — default to no radius (panel meets the viewport edge).
- z-index: backdrop `z-40`/`z-50`, popup `z-50` (above app content; matches existing dialog stacking).

### Backdrop (`Dialog.Backdrop`)
- `fixed inset-0 z-40 bg-black/10 supports-backdrop-filter:backdrop-blur-xs duration-100 data-open:animate-in data-open:fade-in-0 data-closed:animate-out data-closed:fade-out-0` — copy the existing `DialogOverlay` string verbatim.

### Drawer header
- Layout: `flex items-center justify-between p-4 border-b border-sidebar-border`.
- Title: `<DialogPrimitive.Title>` "Your Lists" — `font-heading text-base font-semibold leading-none`.
- Close affordance: `<DialogPrimitive.Close>` rendering a `<Button variant="ghost" size="icon-sm">` with lucide `XIcon` (`size-4`) + `<span className="sr-only">Close</span>` — exact match to the existing dialog close button.

### Phase 9 profile slot (forward-compat — DO NOT FILL)
- Reserve an empty region pinned to the BOTTOM of the drawer column: `<div data-slot="profile-slot" className="mt-auto border-t border-sidebar-border p-4" />` (or render nothing inside; the `mt-auto` + flex-col pushes it to the bottom).
- Leave a code comment: `{/* Phase 9: profile section (avatar / name / sign-out) mounts here — PROF-01/02/03 */}`.
- Do NOT render avatar, name, or sign-out controls. Slot only.

### List rows (`<Link>` per list)
- Container: `<nav className="flex-1 overflow-y-auto py-2">` then one `<Link>` per list.
- Row base: `flex items-center min-h-[48px] px-4 py-3 text-sm truncate transition-colors`.
- Default state: `font-normal text-sidebar-foreground`.
- Hover state: `hover:bg-sidebar-accent` (or `hover:bg-accent`).
- Active state (NAV-02): `aria-current="page"` + `bg-sidebar-accent font-semibold` (background fill AND heavier weight — two redundant cues, one of which is non-color for accessibility). Active match via `useMatch('/list/:code')` → compare `list.share_code === match?.params.code` (D-04; see RESEARCH Pitfall 3 — do NOT use `useParams()` at AppShell level).
- Focus-visible: inherit the row link's default ring; ensure keyboard focus is visible (`focus-visible:bg-sidebar-accent` acceptable).
- Tap behavior (D-08): `onClick={() => onOpenChange(false)}` — navigate via `<Link to={/list/${list.share_code}}>` AND close in one handler. Tapping the already-active row still closes (no-op nav is fine).
- Truncation: long list names `truncate` (matches `LandingPage`).

### Hamburger trigger (in `ListPage` header row)
- Icon: lucide `Menu` (`size-4`/`h-4 w-4`).
- Button: `<Button variant="ghost" size="icon">` (`size-8`), `aria-label="Open navigation"`. Co-located in the existing `ListPage` header `flex items-center justify-between gap-2` row — placed LEADING (left of the list name `h1`), consistent with the rename/delete ghost icon buttons already in that row.
- Tap area ≥44px: the `size-8` visual with the row's `gap-2` spacing is acceptable; if standalone padding is needed, wrap to guarantee ≥44px.
- Focus restoration (RESEARCH Pitfall 6): pass the trigger `ref` as `finalFocus` on `Dialog.Popup` so focus returns to the hamburger on close.

### Animation (tw-animate-css, ~200ms)
- Popup: `duration-200 data-open:animate-in data-open:slide-in-from-left data-closed:animate-out data-closed:slide-out-to-left`.
- Backdrop: `duration-100 ... fade-in-0 / fade-out-0` (matches existing dialog).
- Use `data-open:` / `data-closed:` boolean-attribute variant syntax (Base UI), NOT `data-[state=open]:` (Radix). See RESEARCH Pitfall 4.

### Accessibility contract
- Focus trap, Escape-to-close, `role="dialog"`, `aria-modal`, scroll-lock, backdrop-dismiss: all provided by `Dialog.Root modal` default — do NOT hand-roll.
- Active row: `aria-current="page"`.
- Drawer labelled by its `Dialog.Title` ("Your Lists").
- Focus returns to the hamburger trigger on close (`finalFocus` ref).
