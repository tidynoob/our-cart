<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| ProtectedRoute → AppShell | AppShell only mounts after ProtectedRoute confirms user is authenticated; user is non-null by the time AppShell's useEffect fires |
| AppShell → Supabase (via listsStore) | fetchLists queries owner_id = user.id; Supabase RLS enforces this server-side |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-01 | Information Disclosure | listsStore.fetchLists | mitigate | fetchLists queries .eq('owner_id', userId); Supabase RLS policy enforces owner-scoped reads at DB layer — sidebar cannot show another user's lists |
| T-08-02 | Spoofing | AppShell useEffect (null user guard) | mitigate | useEffect guard: if (user && lists.length === 0) — null user prevents fetchLists call; ProtectedRoute blocks mount until auth resolves, providing belt-and-suspenders |
| T-08-SC | Tampering | npm installs | accept | No new packages installed in this plan — all dependencies (react-router-dom, zustand, lucide-react) already in package.json |
</threat_model>
