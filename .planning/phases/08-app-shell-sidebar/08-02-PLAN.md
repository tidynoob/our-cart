<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → Sidebar (list display) | Sidebar is read-only; no user input accepted in the drawer (no forms, no mutation calls) |
| AppShell → Supabase (via listsStore) | Data already fetched in Plan 01; Sidebar only reads from listsStore.lists passed as props |
| Dialog.Root (modal=true) → page content | Base UI Dialog modal default traps focus inside the drawer, disabling pointer interaction with background content while open |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-03 | Information Disclosure | Sidebar list rows | mitigate | Sidebar receives lists from listsStore which queries eq('owner_id', userId); Supabase RLS enforces owner-scoped reads — other users' lists cannot appear in the drawer |
| T-08-04 | Spoofing / Elevation | Dialog.Root focus trap | mitigate | Dialog.Root modal=true (Base UI default) traps focus inside the open drawer and disables outside pointer interactions — no spoofing of background controls while drawer is open |
| T-08-05 | Elevation | SidebarContext onOpenSidebar | accept | onOpenSidebar only opens the drawer (sets open=true); it carries no auth/data privileges; the callable is scoped to the authenticated AppShell subtree which only mounts after ProtectedRoute confirms auth |
| T-08-SC | Tampering | npm installs | accept | No new packages installed in this plan — zero new dependencies per D-02 |
</threat_model>
