## Spacing Scale

Declared values (multiples of 4). Existing project convention — reuse without change.

| Token | Value | Usage in Phase 9 |
|-------|-------|------------------|
| xs | 4px | Gap between avatar and name text; gap between icon and label (sr-only) |
| sm | 8px | Gap between name row and sign-out button in profile section; button padding |
| md | 16px | Profile slot internal padding (matches existing `p-4` = 16px on the slot) |
| lg | 24px | Gap between sections within the sidebar popup |
| xl | 32px | Not used in Phase 9 additions |
| 2xl | 48px | Not used in Phase 9 additions |
| 3xl | 64px | Not used in Phase 9 additions |

**Exceptions:** All tap targets (sign-out button, pencil edit button, save/cancel buttons, Share2 re-expand button) must meet 44px minimum touch target. Use `min-h-[44px]` or size variants that produce >= 44px when the visual button is smaller (see button spec below).

## Color

All color values are CSS custom properties declared in `src/index.css`. Phase 9 uses only existing tokens.

| Role | Token | Light value | Usage in Phase 9 |
|------|-------|-------------|------------------|
| Dominant (60%) | `--background` | `oklch(1 0 0)` | ListPage main surface (unchanged) |
| Secondary (30%) | `--sidebar` | `oklch(0.985 0 0)` | Sidebar drawer background; profile slot surface |
| Accent (10%) | `--sidebar-accent` | `oklch(0.97 0 0)` | Active list row hover/active state (existing); NOT used for new profile elements |
| Destructive | `--destructive` | `oklch(0.577 0.245 27.325)` | Sign-out button uses `variant="ghost"` with `text-destructive` class — consistent with existing destructive ghost pattern |

**Accent reserved for (this phase):** Sidebar list item hover state and active-list highlight only. The profile section and re-expand affordance do NOT use accent color — they use `ghost` variant (muted hover).

**Attribution badge colors:** Deterministic 2-slot oklch palette from `src/lib/attribution.ts` (blue/green). These are not design system tokens — they are computed. Phase 9 does not change them; they remain the fallback when no avatar `<img>` is available.

**Avatar image:** No border ring or additional color treatment. The `rounded-full` + `h-7 w-7` + `overflow-hidden` on the wrapper clips the img to a circle. Keep it borderless — match the current initials badge visual footprint exactly.

## Component Inventory

### 1. Sidebar Profile Section

**Location:** Fills `<div data-slot="profile-slot" className="mt-auto border-t border-sidebar-border p-4" />` at `Sidebar.tsx:82`.

**Read mode (not editing):**
```
[ Avatar 40px ] Mitchell Griffin       [ Pencil icon ]
[ Sign out button — full width ghost ]
```

| Element | Spec |
|---------|------|
| Avatar img | `h-10 w-10` (40px) circle. `rounded-full overflow-hidden object-cover`. `referrerPolicy="no-referrer"`. `onError` falls back to initials circle (same 40px, `bg-muted text-muted-foreground`, first initial of display name, `text-sm font-semibold`). |
| Display name text | `text-sm font-normal text-sidebar-foreground`. Single line, `truncate`. Sits to the right of avatar, takes remaining flex space. |
| Pencil button | `variant="ghost" size="icon-sm"` (28px visual, `size-7`). `aria-label="Edit display name"`. `Pencil` icon from lucide-react (`h-4 w-4`). Tap target: wrap in a `min-h-[44px]` flex row so the avatar+name+pencil row itself meets 44px. |
| Sign out button | `variant="ghost" size="sm"`. Full width (`w-full`). `text-destructive` class. Label: **"Sign out"**. Placed below the avatar+name row, separated by `mt-2`. No confirmation dialog (D-09). `aria-label="Sign out"`. |
| Row layout | `flex items-center gap-3 mb-2` for avatar + name + pencil. |

**Edit mode (pencil tapped):**

Mirrors ListPage inline list-rename pattern (lines 314-333).

```
[ Input — display name         ] [ Save ] [ Cancel ]
```

| Element | Spec |
|---------|------|
| Input | `h-8 text-sm flex-1`. Placeholder: current display name. `autoFocus`. `aria-label="Display name"`. `onKeyDown`: Enter = save, Escape = cancel. |
| Save button | `variant="outline" size="sm"`. Label: **"Save name"**. Calls `authStore.updateDisplayName(value)` on click. Disabled if trimmed value is empty. |
| Cancel button | `variant="ghost" size="sm"`. Label: **"Cancel"**. Reverts input; exits edit mode. |
| Error state | If `authStore.error` is set after save attempt: `text-xs text-destructive mt-1` below the input row. Text: **"Could not save name. Try again."** Cleared on next save attempt. |
| Layout | `flex items-center gap-2`. Replaces the read-mode row entirely while editing. |

**Transition:** No animation between read and edit mode. Toggle is instant state switch (local `useState` for `isEditing` + `inputValue`).

### 2. AttributionBadge — Avatar Extension

**Location:** `src/components/AttributionBadge.tsx`. Extend existing component; do not replace.

| Element | Spec |
|---------|------|
| Size | Unchanged: `h-7 w-7` (28px) circle. `shrink-0 rounded-full overflow-hidden`. |
| Avatar img (when `avatarUrl` present and not errored) | `<img>` with `src={avatarUrl}`, `alt={name}` (sr-only visible through wrapper `aria-label`), `className="h-full w-full object-cover"`, `referrerPolicy="no-referrer"`, `onError={() => setImgError(true)}`. No border ring. |
| Fallback (no `avatarUrl` OR img error) | Existing deterministic colored-initial badge: `style={{ backgroundColor: color.bg, color: color.text }}`. `text-xs font-semibold`. `getInitials(name)` text. |
| Error fallback trigger | Local `useState(false)` for `imgError`. Guard: only render `<img>` when `avatarUrl && !imgError`. |
| Wrapper | Existing `div` with `flex items-center justify-center`. Remove `style` from wrapper when showing img (apply inline style only to the fallback path). |
| Accessibility | `aria-label={`${name} added this`}` on the wrapper div — unchanged. |
| New prop | `avatarUrl?: string` — optional. No other prop changes. |

**Render logic priority:**
1. `avatarUrl` present AND no img error → render `<img>` inside the circle wrapper
2. Else → render colored initial (existing behavior, unchanged)

### 3. "Show Share Code" Re-Expand Affordance

**Location:** ListPage header row, inside `<div className="flex items-center justify-between gap-2">` (line 302), after `<SyncStatus />`.

**Visibility rule:** Rendered only when `dismissedBanners.has(list.share_code)` is `true`. When the ShareBanner is visible, this button is not rendered.

| Element | Spec |
|---------|------|
| Button | `variant="ghost" size="icon"` (`size-8` = 32px visual, 44px effective touch target via padding on ghost). |
| Icon | `Share2` from lucide-react. `className="h-4 w-4"`. |
| `aria-label` | `"Show share code"` |
| Placement | After `<SyncStatus />` in the header row, before the row's closing tag. Sits at the trailing end of the header row (the row uses `justify-between`, so this trails the title/controls cluster). |
| Action | `onClick={() => restoreBanner(list.share_code)}` — calls `uiStore.restoreBanner`. Banner reappears immediately (Zustand reactive). |

| Action | Confirmation approach |

## Interaction States

### Profile Avatar (40px, sidebar)

| State | Visual |
|-------|--------|
| Default | Google avatar img, `rounded-full`, no border |
| Img load error | 40px circle, `bg-muted`, first initial of display name, `text-sm font-semibold text-muted-foreground` |
| No `avatar_url` in metadata | Same as img load error fallback |

### Attribution Badge Avatar (28px, item row)

| State | Visual |
|-------|--------|
| Default (own item, avatar loaded) | Google avatar img in 28px circle, `rounded-full overflow-hidden` |
| Img load error | Deterministic colored-initial badge (existing blue/green palette) |
| Not own item | Deterministic colored-initial badge from `item.added_by` (existing behavior, unchanged) |
| `added_by` is null | "?" badge (existing behavior, unchanged) |

### Display Name Edit

| State | Visual |
|-------|--------|
| Read mode | Name text + pencil icon button |
| Edit mode | Input (pre-filled) + "Save name" + "Cancel" |
| Saving (async in flight) | Save button disabled; no spinner (instant optimistic update means the store updates before the network call returns) |
| Save error | Error text below input; input remains open for correction |
| Empty input on save attempt | Save button remains disabled; no action |

### Re-Expand Affordance

| State | Visual |
|-------|--------|
| Banner visible | Button not rendered |
| Banner dismissed | `Share2` ghost icon button visible in header trailing position |
| Button pressed | Banner immediately reappears; button disappears |

## Spacing Detail by Component

### Profile Slot (sidebar)

```
border-t border-sidebar-border
p-4 (16px all sides)
  flex items-center gap-3 mb-2      ← avatar + name row (16px = 3 × 4 = gap-3? No: gap-3 = 12px. Use gap-3 for avatar-to-name, which is comfortable)
    h-10 w-10 avatar circle         ← 40px
    flex-1 truncate text-sm name    ← fills remaining width
    size-7 ghost pencil button      ← 28px visual, part of 44px row height
  w-full ghost sm sign-out button   ← mt-2 (8px) above
```

Avatar row min-height: `min-h-[44px]` to satisfy touch target requirement even though the avatar is 40px.

### AttributionBadge (item row)

No layout change to `ItemRow`. The badge footprint stays `h-7 w-7` (28px). The `<img>` fills the wrapper — no additional margin or padding changes.

### Re-Expand Button (header)

`size-8` Button = 32px visual. The header row flex layout provides natural spacing. No margin additions needed — existing `gap-2` on the header row handles separation from `SyncStatus`.

## What Is NOT Changing

To prevent executor scope creep:

- No new color tokens. All colors are existing CSS custom properties.
- No new font sizes or weights. Phase 9 reuses `text-sm`, `text-xs`, `text-base`, `text-2xl`.
- No new layout regions. Profile section fills the pre-built `data-slot="profile-slot"` div.
- No animation on the avatar img load / fallback transition (instant swap on error).
- No spinner on display name save (optimistic update makes it feel instant).
- No toast notifications — inline error text only.
- `NamePromptDialog` is removed, not replaced with a new UI element.
- `AddItemBar` disabled state is removed — no new disabled UI to design for.
