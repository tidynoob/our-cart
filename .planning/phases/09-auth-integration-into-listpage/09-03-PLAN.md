## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user input → Sidebar edit field → authStore.updateDisplayName | User-typed display name; untrusted string |
| sidebar sign-out → authStore.signOut → ProtectedRoute | Auth state change triggers route redirect |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-03-01 | Tampering | display_name XSS in Sidebar | mitigate | resolveDisplayName returns a string rendered via JSX text content — no dangerouslySetInnerHTML anywhere in the profile section. React escapes all string props. |
| T-09-03-02 | Elevation of Privilege | sign-out redirect | mitigate | handleSignOut calls authStore.signOut() which clears user; ProtectedRoute redirects to '/' (hardcoded internal path — no user-supplied URL). |
| T-09-03-03 | Tampering | avatar URL from user_metadata | mitigate | Avatar URL only set from server-controlled user_metadata (populated by Google OAuth at login). User cannot inject arbitrary URLs through the edit flow (only display_name is written). |
| T-09-03-04 | Tampering | Input Validation in Save UI | mitigate | Save button disabled={!editName.trim()} — empty/whitespace saves are rejected in the UI before reaching authStore.updateDisplayName (which also has its own trim guard per D-03). |
| T-09-03-SC | Tampering | npm/pip/cargo installs | accept | No new packages installed this plan. |
</threat_model>
