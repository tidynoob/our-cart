## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-01-01 | Tampering | authStore.updateDisplayName | mitigate | Trim + non-empty guard before calling updateUser (V5 Input Validation). Empty/whitespace strings return early — no DB write. |
| T-09-01-02 | Tampering | supabase.auth.updateUser spread | mitigate | Only `display_name` key is set in `data: { display_name: trimmed }`. Server-controlled keys (full_name, avatar_url) are preserved by Supabase shallow merge. No permission-granting keys are written. |
| T-09-01-03 | Spoofing | USER_UPDATED double-write | accept | USER_UPDATED fires with the server-confirmed user; the existing onAuthStateChange listener idempotently re-sets user. No rollback risk. |
| T-09-01-SC | Tampering | npm/pip/cargo installs | accept | No new packages installed this plan. Zero install surface. |
</threat_model>
