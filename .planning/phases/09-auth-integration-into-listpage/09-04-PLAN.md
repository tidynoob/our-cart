## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user.id → CategorySection → ItemRow | currentUserId flows from auth identity to attribution check |
| list.share_code → restoreBanner | Share code is server-provided; no user injection |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-04-01 | Tampering | resolveDisplayName output → addedBy → DB write | mitigate | resolveDisplayName fallback chain always produces a non-empty string; never passes empty string to addItem (Pitfall 8). The 'Unknown' final fallback ensures addedBy is never ''. |
| T-09-04-02 | Elevation of Privilege | restoreBanner(share_code) | accept | share_code is server-assigned (not user-controlled); restoreBanner only modifies client-side dismissedBanners Set (in-memory, no DB write). No security boundary crossed. |
| T-09-04-03 | Tampering | NamePromptDialog localStorage delete | mitigate | Removing localStorage reads eliminates the possibility of a stale or injected name from localStorage being used as addedBy. Auth display name is the authoritative source. |
| T-09-04-04 | Tampering | attribution XSS (resolveDisplayName → JSX) | mitigate | resolveDisplayName returns a string passed as JSX text / prop value — no innerHTML. React escapes all string outputs. |
| T-09-04-SC | Tampering | npm/pip/cargo installs | accept | No new packages installed this plan. |
</threat_model>
