<objective>
Vertical slice delivering inline edit and delete functionality with a focus-scope approach: tap an item row to edit its name/quantity/category in place, save only when focus leaves the ENTIRE edit row (not individual field blur), delete with trash icon and inline confirmation. The edit row is treated as a single focus unit — clicking between inputs, the Select dropdown, trash icon, or cancel button does not accidentally commit or close the edit.

Purpose: Completes LIST-02 (edit item) and LIST-03 (delete item) requirements. Per D-09, edit mode is inline — fields become editable in the same row. Per D-10, trash icon only appears in edit mode. Per D-11, delete requires inline confirmation with Cancel/Delete buttons before removal.

Addresses review concerns: Inline edit blur/focus interactions (HIGH) — uses focus-scope approach with onMouseDown guards and relatedTarget checks so internal clicks never trigger premature save. Also addresses dirty-check deduping (LOW) — updateItem only called when values actually changed. Edit state kept as local component state in ListPage (not Zustand) per review suggestion.

Output: Updated ItemRow with focus-scope edit mode, new DeleteConfirmation component, ListPage edit/delete state wiring.
</objective>
