## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser event listeners → store actions | visibilitychange and online events trigger fetchItems — events come from browser/OS, not user input; no sanitization needed |
| SyncStatus component → store | Component reads syncStatus as a string enum value; no user-controlled data in this path |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-03-01 | Information Disclosure | SyncStatus revealing connection metadata | accept | SyncStatus exposes only the client's own connection state ('live'/'connecting'/'reconnecting'); no server-side data or partner presence is revealed — OPS-02 presence is explicitly deferred to v2 |
| T-04-03-02 | Denial of Service | Rapid visibilitychange/online events causing fetchItems storm | mitigate | fetchInFlight guard in store's subscribeToList (Plan 02) prevents concurrent duplicate fetchItems calls; guard is cleared in .finally() so it cannot deadlock |
| T-04-03-03 | Tampering | ListPage useEffect cleanup fails to remove event listeners | mitigate | Cleanup function explicitly calls document.removeEventListener('visibilitychange') and window.removeEventListener('online') with the same handler references; verified by ListPage.test.tsx cleanup test |
| T-04-03-04 | Spoofing | supabase_realtime cross-list events reaching wrong subscriber | mitigate | Channel filter `list_id=eq.${listId}` applied server-side; RLS `anon_select_items` policy enforced per event — double defense per D-01 and RESEARCH.md §Security Domain |
| T-04-03-SC | Tampering | npm/pip/cargo installs | accept | No new packages in this plan |
</threat_model>
