SYNC-01 vertical slice — publication enablement + store implementation.

This plan delivers the end-to-end data flow for live sync: the Supabase `items` table is
added to the `supabase_realtime` publication (the blocking prerequisite), and `itemsStore`

in Supabase Studio, and see the merge reducer apply the event to the store — without any UI
wiring (that is Plan 03).

**As a** shopper sharing a grocery list, **I want** changes I make to appear on my partner's
device within 2 seconds, **so that** we never miss or double-buy an item.

Purpose: Makes real-time propagation of mutations (add, edit, check, delete, clear) possible.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Supabase Realtime → Zustand store | Incoming postgres_changes events cross from Supabase WS to the client merge reducer |
| RLS + channel filter → subscriber | anon subscriber only receives events scoped to their list_id |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-02-01 | Information Disclosure | Realtime stream cross-list leakage | mitigate | Channel filter `list_id=eq.${listId}` + RLS `anon_select_items` policy ensure subscriber only receives events for their own list (per RESEARCH.md §Security Domain + D-01) |
| T-04-02-02 | Information Disclosure | DELETE events bypass RLS | accept | Supabase cannot apply RLS to DELETE broadcast; acceptable because both users legitimately own the 2-person list and DELETE payload contains only primary key (no PII) — documented in RESEARCH.md §RLS Interaction with Realtime |
| T-04-02-03 | Information Disclosure | Channel name guessing | accept | Channel name embeds list UUID (`items-${listId}`); UUIDs are 122-bit random — not guessable; publishable key scoped to anon+RLS provides defense-in-depth |
| T-04-02-04 | Spoofing | Malformed event payload clobbering store | mitigate | Merge reducer uses typed cast `(newRow as Item)` for INSERT/UPDATE; DELETE uses `(oldRow as {id?:string}).id` with null guard — invalid payloads produce no-ops or are filtered (per D-03 + RESEARCH.md §DELETE Payload) |
| T-04-02-SC | Tampering | npm/pip/cargo installs | accept | No new packages in this plan; supabase-js already in package.json and audited |
</threat_model>
