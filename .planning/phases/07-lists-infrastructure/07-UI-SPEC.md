## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gaps within button groups |
| sm | 8px | Gap between rename/delete inline buttons; gap inside list row controls |
| md | 16px | Default element spacing; form field gaps; page horizontal padding (`p-4`) |
| lg | 24px | Section gaps between create form and list collection |
| xl | 32px | Vertical padding above page heading on lists-home |
| 2xl | 48px | Empty-state vertical centering padding |
| 3xl | 64px | тАФ (not used in Phase 7) |

Exceptions:
- Touch targets for rename/delete icon buttons: minimum 44px hit area (mobile accessibility). Implemented via `size="icon"` (32px rendered) wrapped with adequate surrounding padding, or use `size="sm"` (28px) inside a `min-h-[44px]` list row container.
- List row minimum height: 48px (`min-h-[48px]`) тАФ mirrors `DeleteConfirmation` existing pattern.

## Color

All tokens are CSS custom properties defined in `src/index.css` (oklch). Do not use raw hex values тАФ use the semantic token names.

| Role | Token | Light Value | Usage |
|------|-------|-------------|-------|
| Dominant (60%) | `--background` | `oklch(1 0 0)` (white) | Page background, list row backgrounds |
| Secondary (30%) | `--muted` / `--secondary` | `oklch(0.97 0 0)` (near-white gray) | Empty-state callout background if used; dialog footer (`bg-muted/50` already in `DialogFooter`) |
| Accent (10%) | `--primary` | `oklch(0.205 0 0)` (near-black) | Primary CTA button ("Create list") only |
| Destructive | `--destructive` | `oklch(0.577 0.245 27.325)` (red) | Delete confirmation button; delete action text |

Accent reserved for: **Primary CTA button only** тАФ the "Create list" submit button. Rename and delete row controls use `variant="ghost"` or `variant="outline"`, not `variant="default"` (primary).

Destructive reserved for: **Delete confirmation button** in the Dialog (`variant="destructive"`) and the delete trigger button on the list row (use `variant="ghost"` with `text-destructive` class тАФ same visual weight as the rename button until confirmed).

Border / separator color: `--border` (`oklch(0.922 0 0)`) тАФ use `divide-y` or `border-b border-border` between list rows.

### Lists-Home Page (`/` тАФ authenticated branch of `LandingPage.tsx`)

Vertical flex column, centered, mobile-first. Mirrors existing `LandingPage` authenticated branch structure.

```
<main class="min-h-screen flex flex-col items-center p-4">

  <!-- Create affordance -->
  <section class="w-full max-w-sm mt-6">

    <CreateListForm />                        <!-- existing, wired to listsStore -->
  </section>

  <!-- Owned lists collection -->
  <section class="w-full max-w-sm mt-6">
    <h2>Your lists</h2>                      <!-- text-xl font-semibold, only rendered when lists.length > 0 -->
    <ul>
      <ListRow /> ├Ч N
    </ul>
    OR
    <EmptyState />                           <!-- when lists.length === 0 and !loading -->
  </section>
</main>
```

Max width: `max-w-sm` (384px) on both the create form and the lists collection тАФ matches existing `LandingPage` pattern exactly.

### List Row

Each owned list renders as a full-width row with the list name as a navigable link and two icon-button controls (rename, delete) on the trailing edge. Plain, no cards, no shadows тАФ per D-02.

```
<li class="flex items-center justify-between min-h-[48px] border-b border-border">
  <a href="/list/:code" class="flex-1 py-3 text-sm font-medium truncate">
    {list.name}
  </a>
  <div class="flex items-center gap-2 pl-2 shrink-0">
    <Button variant="ghost" size="icon-sm" aria-label="Rename {list.name}">
      <Pencil />
    </Button>
    <Button variant="ghost" size="icon-sm" aria-label="Delete {list.name}" class="text-destructive hover:text-destructive">
      <Trash2 />
    </Button>
  </div>
</li>
```

List collection `<ul>` has `divide-y divide-border` тАФ no explicit bottom border on each row needed, `divide-y` handles separation.

### Rename Interaction (inline, within the list row)

When rename is triggered, the `<a>` name link is replaced in-place with an `Input` + confirm/cancel controls. No modal тАФ inline edit matches item-row editing convention from `ListPage`.

```
<li class="flex items-center gap-2 min-h-[48px] border-b border-border py-2">
  <Input
    value={editingName}
    onChange={...}
    onKeyDown={(e) => e.key === 'Enter' && handleRename(); e.key === 'Escape' && cancelRename()}
    aria-label="Rename list"
    class="flex-1 h-7 text-sm"
  />
  <Button variant="outline" size="sm" onClick={handleRename}>Save</Button>
  <Button variant="ghost" size="sm" onClick={cancelRename}>Cancel</Button>
</li>
```

Validation: trim + non-empty before calling `listsStore.renameList`. If empty after trim, show inline error below the input: `<p class="text-xs text-destructive">Name cannot be empty.</p>`. Do not call the store action.

### Delete Confirmation Dialog

Reuses the exact `Dialog` pattern from `ListPage.tsx` lines 318тАУ342 (clear-checked template). No custom modal.

```
<Dialog open={deleteDialogOpen} onOpenChange={setDeleteDialogOpen} disablePointerDismissal>
  <DialogContent showCloseButton={false}>
    <DialogHeader>
      <DialogTitle>Delete '{listName}'?</DialogTitle>
    </DialogHeader>
    <DialogDescription>
      This removes the list and all its items permanently.
    </DialogDescription>
    <DialogFooter>
      <Button variant="outline" onClick={() => setDeleteDialogOpen(false)}>Cancel</Button>
      <Button variant="destructive" onClick={handleDeleteConfirm}>Delete</Button>
    </DialogFooter>
  </DialogContent>
</Dialog>
```

`disablePointerDismissal` is mandatory тАФ prevents accidental backdrop tap on mobile (same rationale as clear-checked dialog per `ListPage.tsx` comment).

## Interaction States

### Lists-Home Loading State

While `listsStore.loading === true`, render a centered spinner in the lists collection area тАФ do NOT hide the create form. Spinner matches existing `LandingPage` loading guard: `animate-spin rounded-full h-8 w-8 border-b-2 border-gray-900`.

```
<div class="flex justify-center py-8">
  <div class="animate-spin rounded-full h-8 w-8 border-b-2 border-gray-900" />
</div>
```

### Error State (lists fetch failed)

If `listsStore.error` is set after fetch, render below the "Your lists" heading:

```
<p class="text-sm text-destructive mt-2" role="alert">
  Could not load your lists. Please refresh.
</p>
```

Do not expose raw Supabase error strings тАФ use this fixed copy only.

### Rename тАФ Pending State

While `renameList` call is in-flight: disable the Save button (`disabled`) and set its text to "SavingтАж". The Input remains interactive (user can keep typing тАФ optimistic update already applied).

### Delete тАФ Pending State

After "Delete" is clicked in the dialog: disable both dialog buttons and change "Delete" button text to "DeletingтАж". The list row disappears optimistically from the page immediately (before await resolves). Dialog closes on success; if error, dialog closes and the row reappears (rollback).

## Accessibility Contract

- All icon-only buttons (`Pencil`, `Trash2`) must have `aria-label` with the list name in context (e.g., "Rename Weekly groceries") тАФ screen readers need context, not just "Rename".
- Rename `Input` receives focus immediately on mount (`autoFocus`) тАФ no extra `useEffect` needed.
- Delete `Dialog`: `disablePointerDismissal` + `showCloseButton={false}` forces explicit button choice. Tab order is Cancel тЖТ Delete (destructive action is last in DOM order per `DialogFooter` layout).
- Empty state: not a live region тАФ it is rendered conditionally when `!loading && lists.length === 0`. No `aria-live` needed.
- Error messages use `role="alert"` for screen reader announcement.
- List collection uses `<ul>` / `<li>` тАФ semantic list markup.
